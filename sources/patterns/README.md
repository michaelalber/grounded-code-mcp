# `patterns` — Design Patterns & Clean Code

Tactical implementation patterns: how to structure code, classes, and
collaborations. Complements `internal` (which covers process) and
`dotnet_csharp` (which covers APIs).

## What belongs here

- Classic design patterns (GoF, POSA)
- Clean code and implementation-level practices
- Dependency injection patterns
- CQRS and event-sourcing patterns
- Object-oriented and functional design
